# hugill-8121.github.io

## 项目设计

    基础css功能包括：
        github提供的静态渲染方案
        基础的github-api功能
            git项目文件（主要为blog文件）的状态获取
            git项目的提交（需要api密钥进行权限登录）
    
    实现功能列表：
        总体思想
            项目可移植；要求在能直接fork到其他项目也可以直接使用，所以要求不出现项目名称等私有化字眼；
            项目第一次进入时即缓存`/blog` 和 `/assert`等文件，已当前的git项目的版本时间作为更新版本号


## web界面设计

### 页头
        博客标题
        博客描述
        右侧为下拉菜单栏，目前功能为：
            登录/登出（主要喂了github-api密钥输入，以保证后续blog的在线更新功能）
                登出后要清除github-api密钥的缓存，保证此时的blog新增/编辑功能失效
            界面刷新
            查询API状态
            新增blog

### 页中

#### blog主页展示界面
- 标签云
    
    主要列举`blog-config.json`中所有列出的tags标签字段，并支持过滤

- blog组
    
    按照`blog-config.json`中组级别对blog进行分组卡片显示

- blog卡片
    
    显示`blog-config.json`中blog `title`,`tags`信息，并显示git 更新人/更新时间/commit信息（已完成）
    批量查询

- blog时间显示过滤器（类似于标签云，但是竖栏实现）

    页中左侧添加一个显示所有blog的月份列表，并支持时间过滤（上述月份左侧有勾选框），页中左侧最下有一个月份范围选择

#### blog单体展示界面
            
- 大写标题
- 标签列表
- 编辑按钮
- git 更新人/更新时间/commit信息（已完成）（本地好像看不到这个，但相关功能已经在github在线blog中测试通过，先不动）
- 正文的markdown渲染（最好是github同种的渲染方案）
- 支持评论区功能

    目前还没想好怎么做，可以做成其他人登录（这个可以用github-api嘛？）按照自定规则提交commit信息到`comment/<blog_uuid>.json`列表中,参数包括：`blog_uuid`，`comment_uuid`,`reply_uuid`,`author`,`time`等，然后渲染，但是这样是不是不好做黑名单之类的，先保留不做
            
#### blog新增/编辑界面

- 页中栏顶部菜单列表
    列出当前打开的所有blog界面列表（与blog_uuid绑定），类似于浏览器那种（切换时保留编辑状态，返回时如果有编辑状态下的blog提醒）
    右键支持常见的固定、关闭、关闭右侧、关闭所有等

- 新增
    
  - 增加文件，按照`blog/<group>/<title>.md`作为blog的git项目保存路径；
  - 生成一个`blog_uuid`
  - 其他和编辑相同
            
- 编辑
  - `group`，`title`不可为空
  - 非新增blog不修改其`blog_uuid`和文件保存路径path
  - 提交时更新文件以及`blog-config.json`文件的更新结果


- 左侧栏（文件栏）

    树形结构显示当前`/blog`文件夹下的所有md文件，方便引用；
    文件夹
        双击收起打开，用 >和v标识
        右键新建文件夹
    文件
        右键功能：鼠标放在某个实际文件上，右键可复制其项目绝对路径（比如/blog/grpc/grpc学习.md）；
        双击功能：打开当前文件，添加到页中顶部菜单列表并打开；

- 右侧栏（资源栏）
    树形结构显示当前`/blog`文件夹下的所有md文件，方便引用；
    文件夹
        双击收起打开，用 >和v标识
        右键
            新建文件夹
            上传文件，需要弹框点击二次按钮确认，直接git 提交，默认commit信息即可；
    资源文件
        双击查看（新建浏览器弹框：比如 https://hugill-8121.github.io/asserts/%E9%BB%91%E7%99%BD%E6%B7%B7.png ）
        右键删除，需要弹框点击二次按钮确认，缓存在浏览器，直接默认commit信息即可；
